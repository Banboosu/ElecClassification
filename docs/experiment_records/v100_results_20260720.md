- 有效三分类数据中没有完全重复序列或冲突标签，未发现直接的重复样本泄漏。
- 远程环境统一为 Python 3.11.15、PyTorch 2.12.1+cu126、CUDA 12.6、cuDNN 9.1.2、
  Tesla V100-PCIE-32GB 和驱动 580.105.08。

基线和 CNN 运行于 Git commit `aa9a396`，正式 TCN 和 MOMENT 运行于 `697cd62`。两者之间的
代码提交主要是 TCN/CNN AMP 稳定性修复；数据与评价协议未改变。当前 CNN 五次运行使用 AMP，
而正式 TCN 使用 FP32。为了让论文主表严格对应同一最终代码和数值精度，建议重新运行一次
FP32 CNN 五随机种子；该组原始训练总耗时约 11.4 分钟，补跑成本较低。

### 3.2 主要结论

1. 当前均值最高的是不做逐序列归一化的 TCN，测试 Macro-F1 为 95.99 ± 0.73%。
2. TCN（none）相对随机森林平均提高 7.09 个百分点，五个 seed 均有提升。
3. CNN 已达到 94.38% Macro-F1，说明局部卷积模式本身非常有效；TCN（none）比 CNN 平均高
   1.61 个百分点，但小样本置信区间跨过 0，不能表述为统计显著优于 CNN。
4. MOMENT v1 线性探测约为 69.46% Macro-F1，但其分类池化未排除 padding，不能据此得出
   “通用时序表征不适合当前任务”的结论；需等待 v2 重跑。
5. 随机森林达到 88.89%，明显高于逻辑回归，说明均值、极值、分位数、斜率和突变等简单统计
   特征已经包含很强的非线性类别信息。论文应保留该基线，否则无法证明深度模型的必要性。

## 4. 归一化消融

验证集最佳 Macro-F1 的五 seed 均值分别为：

| TCN 预处理 | 最佳验证 Macro-F1 | 测试 Macro-F1 |
|---|---:|---:|
| none | **96.08 ± 0.70** | **95.99 ± 0.73** |
| z-score | 95.29 ± 1.25 | 95.02 ± 1.43 |
| minmax | 92.38 ± 1.45 | 92.00 ± 1.37 |

基于同一 seed 的配对差值分析：

| 对比（前者减后者） | Macro-F1 平均差 | 95% t 区间 |
|---|---:|---:|
| TCN none − TCN z-score | +0.97 pp | [-0.55, +2.49] pp |
| TCN z-score − TCN minmax | +3.01 pp | [+0.86, +5.16] pp |
| TCN none − 随机森林 | +7.09 pp | [+6.21, +7.98] pp |
| TCN none − CNN | +1.61 pp | [-0.38, +3.61] pp |

这里只有 5 个 seed，区间只用于描述不确定性，不应写成强统计推断。可以得出：none 的均值和
验证指标最高，minmax 明显较差；但 none 相比 z-score 的约 1 个百分点优势尚不足以声称显著。

逐序列 z-score 和 minmax 都会删除绝对功率幅值，而 raw TCN 的结果最好，说明绝对幅值很可能
是有效判别信息。该信息可能具有真实业务含义，也可能反映设备量程、采集条件或标签规则形成的
捷径。后续建议增加“只用训练集统计量进行全局标准化”的配置：它能保留样本之间的绝对幅值
差异，同时改善数值尺度，是比逐序列归一化更合理的对照。

### 5.1 每类 F1

| 模型 | 类别 0 | 类别 1 | 类别 2 |
|---|---:|---:|---:|
| 随机森林 | 88.88 ± 0.31 | 89.46 ± 0.43 | 88.34 ± 0.29 |
| 1D-CNN（z-score） | 93.60 ± 1.36 | 95.63 ± 1.28 | 93.90 ± 1.27 |
| TCN（none） | **95.60 ± 0.87** | **96.65 ± 0.73** | **95.72 ± 0.71** |
| TCN（minmax） | 90.37 ± 1.51 | 95.21 ± 1.49 | 90.44 ± 1.65 |
| TCN（z-score） | 94.03 ± 1.79 | 96.40 ± 1.31 | 94.62 ± 1.27 |
| MOMENT 线性探测 v1（4 seeds，诊断） | 70.49 ± 0.67 | 73.65 ± 1.80 | 64.23 ± 1.21 |

TCN（none）三个类别的 F1 都超过 95%，类别表现较均衡。minmax 的主要问题是类别 2，五次测试
混淆矩阵合计后的类别 2 Recall 只有 84.64%，大量类别 2 被预测成类别 0。

MOMENT v1 的类别 2 Recall 只有 52.78%，四个测试集累计有 3,482 条类别 2 被预测为类别 0，
另有 1,416 条被预测为类别 1。由于旧分类头混入 padding patch，该现象应作为 v2 的重点核查项，
不能直接归因于冻结表征。

## 6. 训练效率与稳定性

| 模型 | 平均停止 epoch（范围） | 每次平均训练时间 | 峰值显存 | 总参数 | 可训练参数 |
|---|---:|---:|---:|---:|---:|
| 1D-CNN | 37.8（21–50） | 136.3 ± 51.7 秒 | 63 MiB | 51,971 | 51,971 |
| TCN（none） | 30.4（20–37） | 345.5 ± 79.4 秒 | 267 MiB | 218,691 | 218,691 |
| TCN（minmax） | 28.0（20–50） | 321.0 ± 144.5 秒 | 267 MiB | 218,691 | 218,691 |
| TCN（z-score） | 25.0（17–42） | 284.5 ± 115.8 秒 | 267 MiB | 218,691 | 218,691 |
| MOMENT 线性探测 v1（4 seeds） | 26.5（13–34） | 3,091.2 ± 1,090.7 秒 | 1,683 MiB | 341,243,395 | 3,075 |

- TCN 三组正式运行均为 FP32，未触发 AMP 回退，也没有再次出现 NaN/Inf。
- TCN（none）的五次训练总计约 28.8 分钟，TCN（z-score）约 23.7 分钟。
- CNN 五次训练总计约 11.4 分钟。
- MOMENT v1 仅 4 个线性探测就累计约 206 分钟，因为每个 epoch 都重复运行冻结 backbone。
  v2 会为三个 split 各提取一次特征，后续 epoch 只训练 3,075 个分类头参数；V100 配置还为
  partial/full 保持有效 batch 32，并在成功后删除仅用于恢复的 latest checkpoint 以控制磁盘占用。
- 各模型最佳验证 Macro-F1 与测试 Macro-F1 的平均差距均小于 0.4 个百分点，暂未看到严重的
  验证集过拟合；但不同 seed 的停止 epoch 差异较大，保留 early stopping 是必要的。

## 7. 论文可用表述

可在当前阶段写成：

> 在五个随机种子的统一分层划分上，未进行逐序列归一化的 TCN 获得了最高的平均测试
> Macro-F1（95.99% ± 0.73%），相较随机森林统计特征基线提高 7.09 个百分点。逐序列
> z-score 的 TCN 达到 95.02% ± 1.43%，而 minmax 下降至 92.00% ± 1.37%，表明绝对功率
> 幅值可能包含重要类别信息。MOMENT 的旧 v1 分类头没有在最终池化时排除 padding，因此其
> 阶段性数值不纳入正式模型对比，待 mask-aware v2 实验完成后再报告。

当前不应写成：

- “TCN 显著优于 CNN”——平均高 1.61 个百分点，但 5-seed 配对区间跨过 0。
- “不归一化显著优于 z-score”——none 均值更高，但约 0.97 个百分点的差异仍有不确定性。
- “MOMENT 最终效果较差”——旧 v1 pooling 协议存在缺陷，v2 尚未完成。
- “模型具备跨站点或跨时间泛化能力”——当前没有可用的重复设备 ID 或时间戳，只验证了随机
  分层划分下的泛化。

## 8. 后续优先级

1. 停止旧 MOMENT v1 suite，保留现有目录作为诊断记录，不再等待 seed46 或后续微调完成。
2. 使用 mask-aware pooling、冻结特征缓存和轻量 checkpoint 的 v2，从新运行名重新执行策略消融。
3. 用当前 commit 和 `tcn_training.amp: false` 补跑 CNN 五个 seed，形成严格一致的 FP32 主表。
4. 增加训练集全局标准化，并与 none、逐序列 z-score、逐序列 minmax 对比。
5. 保存逐样本预测，重点分析类别 2 被判为类别 0 的曲线，并按序列长度区间报告 Macro-F1。
6. 在取得设备 ID 或时间戳后增加 group split 或 time split；在此之前把随机划分限制写入论文。

## 9. 2026-07-22 MOMENT v2 训练优化实测

本节为单 seed、单 epoch 的远程冒烟和执行效率对照，不纳入论文模型效果主表。三种策略均使用
seed 42、AMP、fused AdamW、mask-aware pooling 和有效 batch 32。

| 策略/执行配置 | 训练耗时 | 验证耗时 | 训练吞吐 | 峰值显存 | 验证 Macro-F1 | 测试 Macro-F1 |
|---|---:|---:|---:|---:|---:|---:|
| 线性探测，冻结特征缓存 | 2.09 s | 0.15 s | 11772.46/s | 1881 MiB（缓存） | 62.90% | 64.34% |
| 部分微调，`32 x 1` | 93.02 s | 10.48 s | 264.13/s | 2692 MiB | 59.20% | 59.95% |
| 完全微调旧执行方式，`16 x 2` + checkpointing | 366.63 s | 10.47 s | 67.01/s | 6030 MiB | 78.30% | 77.95% |
| 完全微调优化方式，`32 x 1` + no checkpointing | 249.00 s | 10.47 s | 98.67/s | 15613 MiB | 78.09% | 77.81% |

线性探测先用 batch 64 缓存 train、validation 和 test 的冻结特征，共耗时 105.94 秒；后续分类头
训练几乎不再受 backbone 吞吐限制。部分微调只解冻最后两个编码层，显存和单轮耗时均较低。

完全微调对照只改变物理 batch、梯度累积和梯度检查点，两个配置的有效 batch 都为 32。优化后
训练吞吐提高 47.2%，训练阶段缩短 32.1%，完整 epoch（含验证）缩短 31.2%；峰值显存占
V100 32GB 的约 47.6%。两次单 epoch 的验证 Macro-F1 相差 0.21 个百分点、测试 Macro-F1
相差 0.14 个百分点，属于冒烟结果，最终效果仍以五随机种子正式实验为准。

若其他 V100 实例出现 OOM，保守回退配置为：

```yaml
training:
  batch_size: 16
  gradient_accumulation_steps: 2
  gradient_checkpointing: true
```

该回退已在同一服务器完整跑通 1 epoch，且保持有效 batch 32。
